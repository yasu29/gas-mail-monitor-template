# GAS Mail Monitor Template

Google Apps Script (GAS) を用いた  
**Gmail監視・スプレッドシート自動記録テンプレート** です。

定期実行（時間トリガー）を前提とし、  
「安全に」「重複せず」「壊れない」ことを重視して設計されています。

---

## できること

* 指定条件でGmailを検索
* 対象メールをスプレッドシートへ自動記録
* 同じメールの二重記録を防止
* 定期実行（例：5分毎）対応
* エラー時の安全停止

---

## 想定している利用シーン

* 請求書メールの自動記録
* 問い合わせメールの一覧化
* 社内通知メールのログ化
* 定型通知メールの自動蓄積

---

## はじめる前に（Prerequisites）

本テンプレートを利用するには：

* Googleアカウント
* Gmail利用可能
* Google Apps Script を使用できる環境
* スプレッドシート編集権限
* 時間トリガー設定権限

特別なAPI設定は不要です。

---

## 使い方（Quick Start）

### 1. スプレッドシートを作成

保存先となるスプレッドシートを作成してください。

シートは2つ必要です：

* データ保存用シート（例：`mail_log`）
* 処理済み管理用シート（例：`_processed`）

---

### 2. config.gs を編集

以下を自分の環境に合わせて変更します。

#### Gmail検索条件

* 差出人
* 件名キーワード
* 検索日数

#### スプレッドシート設定

* spreadsheetId
* 記録先シート名

---

### 3. Apps Script にコードを貼り付け

`src` 配下のファイルを  
新しいApps Scriptプロジェクトに追加してください。

---

### 4. 初回手動実行

`main.gs` の `main()` を実行します。

正常に動けば、対象メールがスプレッドシートに記録されます。

---

### 5. 定期実行（推奨）

Apps Script → トリガー → 時間主導トリガーを設定。

例：

* 5分毎
* 10分毎
* 1時間毎

---

## 処理の流れ（概要）

1. Gmail検索
2. 対象メール取得
3. 既に処理済みか確認
4. 未処理なら記録
5. 処理済みとして登録

同じメールは2回記録されません。

---

## 二重処理防止について

同一メールは `_processed` シートに  
messageId を保存します。

これにより：

* 再実行しても重複しない
* 手動実行と定期実行を併用可能
* 安全に再実行できる

という設計になっています。
